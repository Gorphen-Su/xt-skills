## 上下文

项目目前缺少自动化测试覆盖，导致代码重构和新功能开发时难以保证稳定性。

## 目标 / 非目标

**目标：**
- 建立完整的单元测试框架
- 为核心模块提供至少 80% 的代码覆盖率
- 为关键用户流程添加集成测试

**非目标：**
- 端到端 E2E 测试（由专门的 E2E 测试项目处理）
- 性能基准测试

## 决策

- **测试框架**: 使用 Jest 作为单元测试框架（已项目中使用）
- **测试数据**: 使用工厂函数（factory function）模式生成测试数据
- **模拟**: 使用 jest.mock() 进行模块模拟

**替代方案考虑：**
- Vitest：虽然更快，但项目已深度集成 Jest
- Cypress：用于 E2E，不适合作为单元测试框架

## 风险 / 权衡

- **测试维护成本**: 测试代码需要与业务代码同步维护 → 在代码评审中包含测试审查
- **测试运行时间**: 过多集成测试会拖慢 CI → 将集成测试标记为 slow，按需运行
