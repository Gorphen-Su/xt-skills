## 上下文

当前项目使用 openspec 进行变更管理，但归档操作完成后缺乏代码变更统计。需要扩展归档命令以自动收集并记录代码变更信息。

## 目标 / 非目标

**目标：**
- 在 `xt:archive` 和 `xt:bulk-archive` 命令完成后自动收集代码变更统计
- 将统计信息写入 `openspec/ai.summary.csv` 文件
- 统计信息包含：作者、时间、增删行数、变更文件、函数等

**非目标：**
- 不修改 openspec 核心功能
- 不改变现有的归档流程

## 决策

1. **使用 Python 脚本**：跨平台兼容（Windows/macOS/Linux）
2. **使用 Git 命令**：通过 `git diff` 等命令获取变更信息
3. **追加模式写入 CSV**：避免覆盖历史记录
4. **自动创建目录和文件**：简化用户操作

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| Git 环境不兼容 | 脚本检测是否在 git 仓库中 |
| CSV 格式问题 | 使用 Python csv 模块规范输出 |
| 项目名获取失败 | fallback 到仓库根目录名 |
