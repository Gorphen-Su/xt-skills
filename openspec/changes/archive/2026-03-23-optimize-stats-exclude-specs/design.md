## 上下文

当前 `archive_with_stats.py` 统计所有变更文件的行数，包括 `openspec/` 目录下的 markdown 文件。这些 markdown 文件是 AI 辅助开发过程的规范文档，不属于代码产出。

约束：
- 必须向后兼容现有 CSV 文件
- 新字段需要迁移支持
- 代码统计和规范统计应该独立且清晰

## 目标 / 非目标

**目标：**
- 代码统计排除 `openspec/` 目录下的 markdown 文件
- 新增规范文档统计维度（文件数、行数）
- 保持向后兼容性

**非目标：**
- 不统计其他目录的 markdown 文件（如 README.md、docs/ 等）
- 不修改现有的 token 统计逻辑

## 决策

### 1. 文件过滤策略
**选择**: 在 `collect_stats()` 中按路径前缀过滤，排除 `openspec/` 目录下的 `.md` 文件
**理由**: 简单直接，不影响其他目录的 markdown 文件统计
**替代方案**:
- 配置文件指定排除规则 → 增加复杂度，当前需求简单
- 按文件类型统计分类 → 过度设计

### 2. 规范统计采集方式
**选择**: 单独遍历 `openspec/` 目录下的 `.md` 文件，统计文件数和总行数
**理由**: 与代码统计分离，逻辑清晰
**实现**: 使用 `os.walk()` 遍历目录，统计 `.md` 文件

### 3. CSV 字段扩展
**选择**: 新增 `spec_files` 和 `spec_lines` 两个字段
**理由**: 最小化变更，满足需求
**位置**: 追加到现有字段末尾

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| 现有代码统计数据语义变化 | 更新文档说明，新字段独立不影响旧数据 |
| 排除规则不够灵活 | 当前需求简单，后续可扩展配置化方案 |
